Data Engineering and Analysis — Course Summary

Week 1 Introduction to Data Engineering

General Data Lifecycle (5 Steps)

  1. Generation/Collection — hardware, sensors, text editors
  2. Storage — formats such as JSON, XML, MP4
  3. Maintenance — preprocessing, removing unneeded data
  4. Usage — analyzing preprocessed data for a task
  5. Visualization — presenting results

Data Engineering Lifecycle (5 Stages)

  1. Generation (Source Systems) — where data originates: structured (SQL transactional databases), semi-structured (JSON), or unstructured (IoT device swarms, message queues such as Apache Kafka)
  2. Ingestion — consuming data from sources into a database, governed by the 3Vs: Volume, Velocity, Variety
  3. Storage — secure foundation for downstream processes; modern object storage (e.g., Amazon S3) offers built-in querying such as S3 Select
  4. Transformation — converting and cleansing data (Constructive, Destructive, Aesthetic, Structural)
  5. Serving — final delivery for Analytics, Business Intelligence, or Machine Learning

Types of Data

  • Structured — relational tables
  • Semi-structured — XML, JSON
  • Unstructured — text, streaming data

Origin also splits into Human-generated (emails, videos, social media) and Machine-generated (satellite telemetry, smart meters, CCTV).

Why Data Engineering Matters: The Data Science Hierarchy of Needs

  1. Collect — instrumentation, logging, sensors, external data, user-generated content, Web APIs, scraping
  2. Move/Store — pipelines, ETL, structured/unstructured storage
  3. Explore/Transform — cleaning, anomaly detection, preparation
  4. Aggregate/Label — metrics, segments, features, training data
  5. Learn/Optimize — A/B testing, experimentation, simple ML
  6. AI/Deep Learning — top of the pyramid, requires all prior layers; dirty or siloed data fed directly into AI models fails without the engineering foundation

Goal of Data Engineering

Provide an organized, consistent flow of data supporting analytics, ML, and BI — the bridge between two data types:

  • Operational Data — real-time, transactional data generated by applications
  • Analytical Data — aggregated, optimized data stored in warehouses for reporting

Week 2 Data Collection and Data Sources

Data Collection Methods

  • Observation — watching processes, activities, behaviors
  • Surveys/Polls — behavior, attitudes, opinions
  • Interviews/Focus Groups — structured or open-ended questions
  • Scraping — extracting text/images from websites and social media

Data Sources

Databases, data streams, files.

Data Quality (Six Dimensions)

Accuracy, Completeness, Consistency, Timeliness, Believability, Interpretability.

Data Ethics and Privacy

  • Goal: handle data respecting ethical standards and privacy regulations
  • Protect PII through masking and anonymization
  • Encrypt data both in-flight and at-rest
  • Enforce the Principle of Least Privilege; manage IAM roles

Week 7 Source Systems and Data Ingestion

Source Systems

The starting point where data originates in the data engineering lifecycle. Generate structured data (SQL transactional databases, CSV flat files) or unstructured/semi-structured data.

  • IoT devices / IoT device swarms
  • Smart home sensors
  • Application message queues (e.g., Apache Kafka)
  • Transactional databases

Data Lakes

  • Highly scalable repository holding massive volumes (terabytes/petabytes) of raw, untransformed data
  • Purpose: eliminates data silos — data scattered across disconnected systems (CRMs, analytics tools, ad platforms) — via one unified repository
  • Structure: accepts structured, semi-structured, and unstructured data (unlike data warehouses)
  • Applications: core to ELT pipelines, ingesting data immediately without preprocessing; suited to rapid consolidation and near-real-time analytics

Week 8 Data Pipeline (AWS Glue)

Building an AWS Glue Visual ETL Pipeline

  1. Extract: configure an Amazon S3 Data Source (or Glue Data Catalog source), e.g. reading raw CSV files via the S3 URL

Week 9 Combining Data

Joins (KNIME Joiner Node)

  • Inner Join — intersection only: matching rows in both datasets
  • Left Join — preserves all rows from the left dataset, NULL for unmatched
  • Right Join — preserves all rows from the right dataset, NULL for unmatched
  • Full Outer Join — union of both datasets, NaN/NULL for unmatched columns

Data Lineage

When appending multiple files (e.g., yearly CSVs) into a unified staging table, best practice adds a filesource column to preserve origin tracking.

Week 10 Data Cleaning and Transformation — EDA & Filtering

Exploratory Data Analysis (EDA)

Initial stage of data exploration to maximize insights, uncover structure, extract important variables, and detect anomalies — preparing data for better ML model predictions.

  • Establish research questions
  • Restructure data — create derived variables
  • Create dummy variables — convert categorical data to numerical proxies
  • Graphical tools — histograms for distributions, box plots to compare segments
  • Handle missing observations — mean substitution, regression, multiple imputation
  • Transformation — normalizing variables, logarithmic transforms for skewed distributions

The Four Pillars of Data Transformation

  • Constructive: adding derived features (e.g., deriving Age from Birthdate)
  • Destructive: dropping duplicates, removing missing values, deleting sensitive columns for privacy compliance
  • Aesthetic: standardizing capitalization, country naming, date formats
  • Structural: splitting/reorganizing columns, optimizing layout for BI tools

Data can also be grouped via Aggregation (e.g., Total Spent per user across orders).

Statistics (KNIME Statistics View)

  • Calculates minimum, maximum, variance, standard deviation, and quantiles (25%, 50%/Median, 75%)
  • Automatically ignores missing values and NaN

Filtering Data

  • Column Filter: exclude columns manually, by data type, by regex, or by wildcard
  • Row Filter: filters by attribute values, ranges, or missing values

Week 11 Data Cleaning and Transformation — Normalization, Missing Data, Outliers

"Garbage In, Garbage Out"

Models trained on dirty or bad data are poorly trained and produce irrelevant analysis.

Normalization vs. Standardization

MethodEffect on ShapeEffect on OutliersBest For
Min-Max Normalization (0.0–1.0)PreservedNot reducedImage processing, KNN
Standardization (Z-score, mean 0, std 1)ChangedReducedPCA, linear models

KNIME's Denormalizer node reverts these transformations to original values.

Causes of Missing Data

  • Equipment malfunctions
  • Deletion due to inconsistency with other records
  • Data entry misunderstandings
  • Data not considered important at time of entry

Handling Missing Data

Dropping records loses information; imputation is preferred:

  • Global constant, mean, median (robust against outliers), or mode (categorical)
  • Forward/Backward Fill: use preceding or following value
  • Linear Interpolation: estimate from neighboring points
  • Moving Average: average of surrounding points
  • Regression Imputation: ML regression predicts the missing value

Noisy Data

Binning: sort the data, partition into equal-frequency bins, smooth using bin means, medians, or boundaries. Clustering and regression functions can also smooth noise.

Outlier Detection

  • Clustering methods (e.g., DBSCAN)
  • Box Plots — points beyond 1.5× the Interquartile Range (IQR) are flagged as outliers
  • Z-Score: flag values with Z-score greater than 2 or less than -2 standard deviations from the mean
Normal Distribution: mean, median, and mode are equal.

Empirical Rule:
68% within ±1 standard deviation
95% within ±2 standard deviations
99.7% within ±3 standard deviations

Week 13 Big Data Technologies

Big Data Characteristics (3Vs)

  • Volume: large-scale data accumulation
  • Velocity: rapid generation/processing for real-time analytics
  • Variety: structured tables, text, XML, graph data

Batch vs. Stream Processing

TypeData TemperatureMechanism
Batch ProcessingCold (infrequently accessed)Scheduled intervals, large volumes (Hadoop)
Stream ProcessingHot (frequently accessed)Near real-time, sequential (Kinesis)

Hadoop Ecosystem

  • HDFS: NameNode manages metadata; DataNodes store and replicate data blocks
  • YARN: resource management across the cluster
  • MapReduce: iterates over records, splits and maps them to keys, reduces into aggregated totals

Amazon EMR

Processes large logs via Hadoop frameworks. Cluster roles: Main Nodes, Core Nodes, Task Nodes.

Tool Reference Cross-Week — KNIME & AWS Mechanics

KNIME Node Behavior

  • GroupBy: aggregation — Mean for numeric columns, Mode for categorical columns
  • Value Lookup: dictionary-style match; on no match — insert missing value, match next smaller, or match next larger
  • Concatenate: stacks tables on top of each other (appends rows)
  • Database Connections: SQLite Connector, DB Query Reader for direct SQL execution
  • CSV Writer: Overwrite existing file, Append to bottom of it, or Fail to prevent unintentional overwriting
  • Node Status Colors: Red (not configured), Yellow (configured, ready), Green (executed), Red X (error)

AWS Glue (General)

Cloud-based ETL: extract from S3 or Data Catalog, change schemas, filter records (SQL-like expressions with AND/OR), convert formats (CSV → Parquet/GZIP).
